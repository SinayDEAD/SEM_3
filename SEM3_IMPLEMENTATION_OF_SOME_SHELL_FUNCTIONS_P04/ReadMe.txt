Задание: P04 

Вариант:

1. cp old-file new-file

2. grep (-v) [pattern] [file/по умолчанию 1]

3. cat (-n) filenameS/1
_________________________________________________________________________________________

1. Про cp:
   Задача - копировать файл с вариантом, что на конце обязательно новое имя.
   Из особенностей - проблема может возникнуть с правами доступа, свято верить,
   что всё просто, нельзя, вдруг что..., но решение через chmod. Если файл нельзя
   Открыть даже на чтение, то выдал ошибку и выйдет.
   Спокойно копирует с директорий и без, тогда остаётся в той же.
   Опять же, ограничений на размер файла нет.

2. Про grep:
   Задача - отлавливать в строке первое вхождение паттерна и выводить (тут с подсветкой).
   Моя реализация при ключе -v просто выводит строки без вхождения паттерна. 
   Из плюсов, grep ловит вхождение в строке даже очень большой длинны и спокойно выводит 
   её и в обратном случае тоже. 
  
   Подзадачи:
   
   2.1. void do_grep(int fd, const char *const filename,
             regex_t *const pattern, const int use_ansi); 
   2.2  void do_anti_grep(int fd, const char *const filename,
                  regex_t *const pattern, const int use_ansi);

    -Первый без ключа и с подсветкой.
    -Второй с ключом и без неё.


3. Про cat:
   Задача - cat это сокращения от слова catenate. По сути, задача команды cat очень 
   проста - она читает данные из файла или стандартного ввода и выводит их на экран.
   Если есть ключ (-n) выводит с нумерацией строк. Все возможности обычного в этой
   реализации сохранены. Точно также как и в grep ограничений на размер файла у 
   команды нет. Нумерация не сквозная, кот на Mac OS нумеруется отдельно каждый файл.
 
   Подзадачи:

   3.1 void StrokiVisovCat(char *argv[]);          
   3.2 void VisovCat(char *argv[]);
   
   - Первое это кот с ключом нумерации строк 
   - Второе это кот без ключа и спокойный написанием
